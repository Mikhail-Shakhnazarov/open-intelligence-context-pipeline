# Contributing

Contributions are welcome if they improve clarity, testability, and real-world usability.

## Contribution stance

- Prefer small, composable changes.
- Keep text ASCII-only (no smart quotes/dashes/arrows) for transport compatibility.
- Preserve the protocol's core invariants:
  - bounded packets,
  - trace alongside payload,
  - proposals are non-binding until ratified,
  - explicit boundary enforcement.

## Suggested contribution types

- additional worked examples (brief + receipt + change requests),
- workplace boundary patterns,
- conformance fixtures and checklists,
- improvements to starter-kit templates.
