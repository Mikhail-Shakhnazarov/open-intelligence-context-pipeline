## Overview

Procedures are named transforms with acceptance criteria. They define how messy inputs become governed artifacts.

## Procedure set (plain language)

Ingest (quarantine inbox):
- capture raw material with source metadata and boundary class.

Normalize (normalized record; canonical record):
- convert a raw capture into a stable reference representation that preserves meaning and provenance.

Derive (rebuildable indexes; derived caches):
- build summaries/embeddings/indices as convenience layers that can be regenerated.

Build packet (context packet set):
- produce a packet + selection log + update proposals for a task under declared boundaries and a selection policy.

Propose updates (update proposals):
- write candidate durable updates with derivation marking and evidence pointers.

Approve (ratification; promotion):
- explicitly accept proposals into durable registries with recorded rationale.

Exact acceptance criteria are defined in the normative edition.
